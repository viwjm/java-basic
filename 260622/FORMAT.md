# CVID — 문자 동영상 포맷 (Character Video Format)

cmd 창에 문자(글자)의 밝기 차이를 이용해 동영상을 재생하기 위한
아주 단순한 텍스트 기반 동영상 포맷입니다.

- **포맷 이름**: CVID (Character Video)
- **확장자**: `.cvid`
- **인코딩**: UTF-8 (실제 내용은 ASCII 숫자/기호뿐)

---

## 1. 전체 구조

```
<가로>*<세로>\n
<밝기값>,<밝기값>,<밝기값>, ... ,<밝기값>
```

파일은 크게 **헤더(첫 줄)** 와 **본문(나머지 전체)** 두 부분으로 나뉩니다.

---

## 2. 헤더 (첫 번째 줄)

- 이 동영상이 **몇 바이 몇** 인지를 나타냅니다.
- 형식: `가로*세로` (가로 픽셀 수, 별표 `*`, 세로 픽셀 수)
- 예: `120*100` → 가로 120칸, 세로 100칸

헤더 줄은 `\n` (줄바꿈) 으로 끝납니다.

---

## 3. 본문 (두 번째 줄부터 끝까지)

- 쉼표(`,`)로 구분된 **정수 밝기값**들의 나열입니다.
- 각 밝기값은 **0 이상 99 이하**의 정수입니다.
- 숫자가 **클수록 밝고**, **작을수록 어둡습니다**.
- 값들이 나열되는 순서:
  1. **첫 프레임부터** 순서대로
  2. 한 프레임 안에서는 **맨 윗 줄(행) 부터** 아래로
  3. 한 줄 안에서는 **왼쪽부터** 오른쪽으로
- **줄(행)과 프레임의 구분 기호는 없습니다.** 오직 쉼표로만 구분되며,
  플레이어가 `가로 * 세로` 값으로 알아서 끊어 읽습니다.

### 픽셀/프레임 개수 계산

- 한 프레임의 픽셀 수 = `가로 × 세로`
- 전체 밝기값 개수는 반드시 `(가로 × 세로)` 의 배수여야 합니다.
- 프레임 수 = `전체 밝기값 개수 ÷ (가로 × 세로)`

---

## 4. 밝기값 → 문자 매핑

플레이어는 밝기값(0~99)을 아래 10단계 문자 램프(ramp)로 변환해 출력합니다.
왼쪽이 가장 어둡고(빈 공간), 오른쪽으로 갈수록 밝습니다(꽉 찬 글자).

| 밝기값 범위 | 문자 |
|:----------:|:----:|
| 0 ~ 9      | (공백) |
| 10 ~ 19    | `.`  |
| 20 ~ 29    | `:`  |
| 30 ~ 39    | `-`  |
| 40 ~ 49    | `=`  |
| 50 ~ 59    | `+`  |
| 60 ~ 69    | `*`  |
| 70 ~ 79    | `#`  |
| 80 ~ 89    | `%`  |
| 90 ~ 99    | `@`  |

> 램프 문자열: `" .:-=+*#%@"` (어두움 → 밝음)

---

## 5. 예시

`3*2` (가로 3, 세로 2) 크기의 동영상이고 프레임이 2개인 경우:

```
3*2
0,50,99,99,50,0,99,99,99,0,0,0
```

- 한 프레임의 픽셀 수 = 3 × 2 = 6
- 전체 숫자 12개 → 프레임 수 = 12 ÷ 6 = 2

**프레임 0**
```
0  50 99   →  "  +@"  (공백, +, @)
99 50 0    →  "@+  "  (@, +, 공백)
```

**프레임 1**
```
99 99 99   →  "@@@"
 0  0  0   →  "   " (공백 3개)
```

---

## 6. 재생 방식 (플레이어)

- `while (true)` 로 전체 프레임을 **무한 반복** 재생합니다.
- 한 프레임을 그린 뒤 `Thread.sleep(100)` (0.1초) 쉽니다 → 초당 약 10프레임.
- 매 프레임을 그리기 전에 ANSI 이스케이프 코드로 화면을 정리합니다.
  - `\033[2J` : 화면 전체를 지움 (clear screen)
  - `\033[H`  : 커서를 좌측 상단(home)으로 이동
  - 이 둘을 쓰면 프레임이 아래로 흘러내리지 않고 같은 자리에 덮어써져
    애니메이션처럼 보입니다.

java Player.java "C:\경로\영상파일.cvid"
