# 📌 문제 이름

프로그래머스 181855번 - 문자열 묶기

## 🔗 문제 링크

https://school.programmers.co.kr/learn/courses/30/lessons/181855

## 💡 문제 해결 전략

이건 벌써 뭔가 최적화의 여지가 많을 것 같은 문제이다. 
재미있는 부분은 알파벳이 무엇인가는 중요하지 않다는 것이다.
["a", "b"] 이것과 ["a", "a"] 이것은 똑같다.
그치만 뭘 해도 한 번 순회는 해야 할 것같아서 최적화가 큰 의미는 없는 것 같기도 하다.
1 ≤ strArr의 원소의 길이 ≤ 30 니까 일단 종류는 최대 30개다.
그리고 하나가 최대 10만 개 있을 수 있다.
그냥 세자 멋있는 방법을 못 찾아 아쉽다.
배운 걸로만 하려고 아래와 같이 짰지만 사실은 Math.max를 쓰는 게 낫다.

## 💻 코드

```java
class Solution {
    public int solution(String[] strArr) {
        int max = 0;
        int[] counter = new int[30];
        
        for (String str : strArr) {
            counter[str.length() - 1]++;
            max = max > counter[str.length() - 1] ? max : counter[str.length() - 1];
        }
        
        return max;
    }
}
```
